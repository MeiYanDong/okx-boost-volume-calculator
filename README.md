# OKX Boost 交易量计算器

这是一个本地网页工具，用来帮你估算 **OKX Boost 活动里的 10 日平均交易量**。

你可以把它理解成一个“自动查账 + 自动算分”的小工具：输入钱包地址和快照日期，它会去链上找这个钱包在 BNB Chain 上通过 OKX DEX 聚合器完成的交易，然后按 Boost 规则算出交易量。

## 适合谁用

适合想参加 OKX Boost 活动，但不想手动翻交易记录、手动算 10 天平均值的人。

如果你是第一次接触链上数据，也不用怕。你只需要知道三件事：

1. 钱包地址就是你参加活动的钱包。
2. 快照日期就是活动检查你是否达标的那一天。
3. 结果里的 Boost 交易量越高，越接近活动要求。

## 怎么使用

先启动工具：

```bash
npm install
npm run dev
```

然后打开：

```text
http://localhost:5173/
```

页面上只需要填两个东西：

1. 钱包地址
2. 快照日期

填好之后点击 **扫描 BNB OKX 聚合交易**。

算完以后，页面会展示：

- Boost 交易量：10 天平均后的核心结果
- 10 日合计：这 10 天每天 Boost 交易量加起来的总数
- 合格成交额：链上能识别出来、符合当前规则的成交额
- 交易笔数：本次扫描找到并参与统计的交易数量
- 每日数据：每天分别贡献了多少交易量
- 交易明细：每一笔交易的 hash、交易对、基础倍数、额外加成和 Boost 结果

## 快照日期怎么理解

假设你选择的快照日期是 `2026-05-03`，工具会计算：

```text
2026-04-24 UTC 00:00
到
2026-05-04 UTC 00:00
```

也就是包含 `2026-05-03` 当天在内的 10 个 UTC 日期。

如果你选择今天，而今天还没结束，结果会显示为实时预估。意思是：工具会先算到当前最新区块，等今天结束后数字可能还会变化。

## 代币额外加成怎么填

有些活动会给某些代币额外加成，比如 `+50%`。这时你不需要重新扫链。

正确流程是：

1. 先扫描一次。
2. 等结果区出现“代币额外加成”。
3. 找到有活动加成的代币。
4. 在它旁边填百分比，比如 `50`。

填完以后，结果会立即在本地更新，不会重新扫描链上数据。

注意：代币额外加成不是筛选条件。也就是说，就算你给某个代币填了加成，其他 OKX 聚合交易仍然会参与计算。

## 当前统计范围

当前版本只统计：

```text
BNB Chain 上的 OKX DEX Router 交易
```

暂时不统计：

- Base
- Arbitrum
- Solana
- 其他链上的 OKX 钱包交易

所以，如果你在 OKX 钱包里看到某笔交易有 Boost 交易量，但这个工具没有算到，先确认那笔交易是不是发生在 BNB Chain。之前我们已经遇到过一种情况：OKX 钱包里显示的是 OKX DEX Aggregator，但交易实际发生在 Base 链，这种交易当前版本不会纳入。

## 为什么不用用户填写 API Key

用户界面不展示这些东西：

- Ankr Advanced API
- Etherscan V2 API Key
- 交易记录 CSV

原因很简单：这些是工具的数据源配置，不是普通用户该关心的输入项。

现在浏览器只访问本地同源接口：

- `/api/rpc`
- `/api/ankr`
- `/api/explorer`

真正的 RPC URL 和 API Key 都留在本地 Node 服务里，不会被打包进前端页面。

## 本地配置

如果你只是使用别人已经启动好的页面，可以跳过这一节。

如果你自己在本地运行，需要准备 `.env.local`。可以从 `.env.example` 复制一份：

```bash
cp .env.example .env.local
```

推荐配置 Ankr Advanced API：

```bash
ANKR_MULTICHAIN_RPC_URL=https://rpc.ankr.com/multichain/你的_token
```

如果你只有普通 BSC RPC，也可以填：

```bash
BSC_RPC_URL=你的_BSC_RPC
```

如果你有可用的 Etherscan API V2 Key，也可以填：

```bash
ETHERSCAN_API_KEY=你的_key
```

修改 `.env.local` 后，需要重启：

```bash
npm run dev
```

## 计算口径

当前 V1 的口径如下：

- 链：BNB Smart Chain
- 时间窗口：快照日期 `D` 往前数 9 天，再加上 `D` 当天，共 10 个 UTC 日期
- 平均方式：10 天合计 Boost 交易量除以 10，没交易的日期也算 0
- 单笔公式：`稳定币侧成交额 × 程序自动判断的基础倍数 × 用户填写的代币额外加成`
- 数据缓存：同一个钱包和快照日期算过以后会保存在本地，刷新页面会自动恢复

## 安全设计

这个工具已经做了几层基础保护：

- API Key 不进入前端 bundle。
- 浏览器只能请求同源的 `/api/*`。
- `/api/rpc` 只允许读取链上数据，不允许发交易。
- `/api/ankr` 只允许查询 BSC 钱包交易。
- `/api/explorer` 只允许查询 BSC 的 `account/txlist`。
- 生产模式会加基础安全响应头，比如 CSP、Referrer-Policy、Permissions-Policy。

这解决的是“不要把付费 API Key 暴露给浏览器”的问题。

如果以后要公开部署给很多人用，还需要继续加：

- 登录鉴权
- 请求限流
- 服务端缓存
- 更完整的错误日志和监控

## 常见问题

**为什么我在 OKX 钱包里看到交易量，但这里没算到？**

先看那笔交易是不是 BNB Chain。如果是 Base、Arbitrum 或 Solana，当前版本不会统计。

**为什么结果和 OKX 页面差一点点？**

当前工具主要用链上稳定币数量估算 USD。OKX 页面可能使用自己的历史价格和内部口径，所以小数级差异是正常的。

**为什么第一次扫描比较慢？**

第一次需要查链上数据。算过一次以后，同一个钱包和快照日期会有本地缓存，刷新页面会自动恢复。

**点刷新按钮会发生什么？**

刷新按钮会清理本地缓存和本地保存的计算结果。下次需要重新扫描链上数据。
