# OKX Boost 交易量计算器

这份说明是给用户看的。

你不需要自己运行命令，也不需要研究项目怎么实现。你只需要把下面的话交给你的 Agent，让它帮你把工具跑起来，并完成交易量计算。

## 直接复制给你的 Agent

你可以把这一段发给你的 Agent：

```text
请帮我运行这个项目，并用它计算我的 OKX Boost 交易量：

GitHub 项目：https://github.com/MeiYanDong/okx-boost-volume-calculator

请你完成这些事：
1. 把项目拉到本地。
2. 按项目需要安装依赖并启动本地网页。
3. 如果项目需要数据源配置，请在本地配置，不要把密钥写进前端页面，也不要提交密钥文件。
4. 打开本地页面后，帮我输入钱包地址和快照日期。
5. 点击“扫描 BNB OKX 聚合交易”。
6. 如果页面出现“代币额外加成”，根据我告诉你的活动规则填写百分比。
7. 最后告诉我 Boost 10 日平均交易量、10 日合计交易量、统计窗口、识别到的交易笔数，以及有没有警告。

注意：当前工具只统计 BNB Chain。不要把 Base、Arbitrum、Solana 或其他链的交易算进去。
```

## 你需要提前准备什么

发给 Agent 前，最好准备好这几项：

1. 钱包地址
2. 快照日期
3. 是否有代币额外加成
4. 如果有目标线，比如需要达到多少 Boost 交易量，也一起告诉 Agent

如果你不知道某个代币有没有额外加成，可以先让 Agent 跑一次。工具会在结果里列出这次交易涉及的代币，你再决定哪些要填加成。

## 你应该怎么告诉 Agent 钱包信息

可以这样说：

```text
我的钱包地址是：0x...
快照日期是：2026-05-03
如果结果里出现某某代币，它有额外 +50% 加成。
请帮我判断当前是否达标。
```

如果你还没有确定快照日期，可以这样说：

```text
请先帮我确认这个活动应该选择哪一天作为快照日期，再用工具计算。
不要直接猜日期。如果不确定，请先问我。
```

## Agent 跑完后应该给你什么结果

你要让 Agent 最后用人话汇报，而不是只截图或只说“完成了”。

一份合格的结果应该包含：

1. Boost 10 日平均交易量
2. 10 日合计交易量
3. 统计窗口是哪几天
4. 识别到多少笔交易
5. 哪些代币填写了额外加成
6. 有没有警告或解析失败
7. 如果你给了目标线，还要告诉你是否达标、差多少或超多少

你可以要求 Agent 用这种格式回复：

```text
计算结果：
- Boost 10 日平均交易量：
- 10 日合计交易量：
- 统计窗口：
- 交易笔数：
- 已应用的额外加成：
- 警告：
- 是否达标：
```

## 快照日期是什么意思

快照日期会被包含在统计窗口里。

例如选择 5 月 3 日，就是看包含 5 月 3 日在内的过去 10 个 UTC 日期。

如果选择今天，而今天还没有结束，结果只是实时预估。你要让 Agent 明确提醒你：今天后面如果继续交易，结果还会变化。

## 额外加成怎么交代给 Agent

有些活动会给某些代币额外加成，比如 +50%。

你可以这样告诉 Agent：

```text
如果结果里有 RAVE，RAVE 有额外 +50% 加成。
如果没有这个代币，就不用填。
```

或者：

```text
我不确定哪些代币有加成。你先跑一次，把识别到的代币列给我，我再告诉你哪些需要加成。
```

注意：额外加成不是筛选条件。它只是倍率。填了某个代币加成之后，其他 OKX 聚合交易仍然要参与计算。

## 当前工具的边界

当前工具只统计：

```text
BNB Chain 上的 OKX DEX Router 交易
```

当前工具暂不统计：

- Base
- Arbitrum
- Solana
- 其他链上的 OKX 钱包交易

如果你在 OKX 钱包里看到某笔交易有 Boost 交易量，但工具没有算到，请把交易 hash 发给 Agent，让它先判断这笔交易在哪条链上。

如果交易实际发生在 Base 或其他链，这不是当前工具漏算 BNB，而是当前版本还没支持多链。

## 如果结果和 OKX 页面不完全一样

小幅差异是正常的。

可以让 Agent 这样解释：

```text
这个工具主要根据链上稳定币侧成交额估算 USD。OKX 页面可能使用自己的历史价格和内部口径，所以小数级差异是正常的。
```

如果差异很大，让 Agent 继续检查：

1. 快照日期是否选错
2. 交易是否发生在其他链
3. 代币额外加成是否漏填
4. 页面是否有警告或未解析交易

## 你不需要做什么

你不需要自己填写 API Key。

你不需要自己导入 CSV。

你不需要自己运行项目命令。

你不需要理解内部实现。

你只需要把钱包地址、快照日期、活动加成规则告诉 Agent，然后让 Agent 按这份说明操作。
