# OKX Boost 交易量计算器

如果你正在参加 OKX Boost 活动，最麻烦的事情通常不是交易本身，而是算清楚：

我现在到底有多少 Boost 交易量？  
过去 10 天平均是多少？  
明天快照时我够不够资格？

这个工具就是为了解决这个问题。

你不用自己翻区块浏览器，不用手动整理交易记录，也不用拿计算器一笔笔加。你只需要把钱包地址和快照日期告诉 Agent，让 Agent 把这个项目跑起来，工具会自动扫描并计算结果。

## 这个工具解决什么痛点

OKX Boost 的交易量计算对新手很不友好，主要有几个坑：

1. **不是看单日交易量，而是看过去 10 天平均值。**
   很多人只看今天刷了多少，但活动真正关心的是一个时间窗口里的平均表现。

2. **快照日期容易理解错。**
   快照日期当天是包含在统计里的，不是只算到前一天。

3. **交易记录分散在链上。**
   一笔交易看起来是 OKX 聚合交易，但你还要确认它是不是在当前工具支持的链上。

4. **不同代币可能有额外加成。**
   有些代币可能 +50%，有些没有。如果手算，很容易漏掉或算错。

5. **OKX 钱包看到的交易量不一定都属于 BNB Chain。**
   当前工具只统计 BNB Chain。如果某笔交易发生在 Base、Arbitrum 或 Solana，当前版本不会纳入。

## 这个工具怎么帮你解放出来

你把任务交给 Agent 后，Agent 可以帮你完成这些事：

- 打开并运行这个项目
- 在页面里输入钱包地址和快照日期
- 自动扫描 BNB Chain 上的 OKX DEX 聚合交易
- 自动计算每笔交易的基础倍数
- 根据你提供的活动规则填写代币额外加成
- 汇总出 10 日平均 Boost 交易量
- 检查有没有警告、漏算风险或跨链边界

也就是说，你不用亲自处理链上数据。你只需要告诉 Agent：“这是我的钱包，这是快照日期，这是代币加成规则，请帮我算并告诉我是否达标。”

## 最终你会得到什么结果

一次完整计算后，你应该拿到这些信息：

1. **Boost 10 日平均交易量**  
   这是最核心的数字，用来判断你是否接近活动要求。

2. **10 日合计交易量**  
   这是 10 天每天 Boost 交易量加起来的总数。

3. **统计窗口**  
   告诉你这次到底算了哪 10 个 UTC 日期。

4. **交易笔数**  
   告诉你工具识别到了多少笔 BNB Chain OKX 聚合交易。

5. **代币额外加成**  
   告诉你哪些代币被填写了额外加成，比如 +50%。

6. **警告信息**  
   如果有交易无法解析、日期还没结束、或者某些交易不在支持范围内，Agent 应该提醒你。

如果你告诉 Agent 一个目标线，比如“需要达到 100,000 美元 Boost 交易量”，Agent 还应该告诉你：已经达标、还差多少，或者超了多少。

## 你需要准备什么

在让 Agent 操作前，你最好准备好：

1. 钱包地址
2. 快照日期
3. 活动目标线，如果你知道的话
4. 哪些代币有额外加成，如果你知道的话

如果你不知道哪些代币有加成，也没关系。可以先让 Agent 跑一次，让它把结果里出现的代币列出来，你再判断哪些代币需要加成。

## 如何让 Agent 把项目跑起来

你不需要自己运行命令。直接把下面这段话复制给你的 Agent：

```text
请帮我运行这个项目，并用它计算我的 OKX Boost 交易量：

GitHub 项目：https://github.com/MeiYanDong/okx-boost-volume-calculator

请你完成这些事：
1. 把项目拉到本地。
2. 根据项目实际情况安装依赖、配置本地数据源，并启动本地网页。
3. 如果需要配置 RPC 或 API Key，请只在本地配置，不要暴露到前端页面，不要提交密钥文件。
4. 打开本地页面后，帮我输入钱包地址和快照日期。
5. 点击“扫描 BNB OKX 聚合交易”。
6. 如果页面出现“代币额外加成”，根据我告诉你的活动规则填写百分比。
7. 最后用人话告诉我 Boost 10 日平均交易量、10 日合计交易量、统计窗口、交易笔数、已应用的额外加成，以及有没有警告。

注意：当前工具只统计 BNB Chain。不要把 Base、Arbitrum、Solana 或其他链的交易算进去。
```

然后再补充你的个人信息：

```text
我的钱包地址是：0x...
快照日期是：2026-05-03
如果结果里出现 RAVE，RAVE 有额外 +50% 加成。
我的目标线是：xxxx 美元 Boost 交易量。
请帮我判断当前是否达标。
```


如果 Agent 只说“完成了”，那是不够的。你需要它把关键数字讲清楚。

## 快照日期怎么理解

快照日期会被包含在统计窗口里。

比如选择 5 月 3 日，就是看包含 5 月 3 日在内的过去 10 个 UTC 日期。

如果选择今天，而今天还没有结束，结果只是实时预估。今天后面如果继续交易，数字还会变化。

所以你要让 Agent 明确区分：

- 已结束日期的最终结果
- 今天这种尚未结束日期的实时预估

## 代币额外加成怎么告诉 Agent

如果某个代币有额外加成，比如 +50%，你可以这样说：

```text
如果结果里有 RAVE，RAVE 有额外 +50% 加成。
如果没有 RAVE，就不用填。
```

如果你不确定：

```text
我不确定哪些代币有加成。你先跑一次，把识别到的代币列给我，我再告诉你哪些需要加成。
```

注意：额外加成只是倍率，不是筛选条件。填了某个代币加成之后，其他 OKX 聚合交易仍然要参与计算。

## 当前工具的边界

当前版本只统计：

```text
BNB Chain 上的 OKX DEX Router 交易
```

当前版本暂不统计：

- Base
- Arbitrum
- Solana
- 其他链上的 OKX 钱包交易

如果你在 OKX 钱包里看到某笔交易有 Boost 交易量，但工具没有算到，请把交易 hash 发给 Agent，让它先判断这笔交易在哪条链上。

如果交易实际发生在 Base 或其他链，这不是当前工具漏算 BNB，而是当前版本还没支持多链。

## 如果结果和 OKX 页面不完全一样

小幅差异是正常的。

原因是：这个工具主要根据链上稳定币侧成交额估算 USD；OKX 页面可能使用自己的历史价格和内部口径，所以小数级差异可能存在。

如果差异很大，让 Agent 继续检查：

1. 快照日期是否选错
2. 交易是否发生在其他链
3. 代币额外加成是否漏填
4. 页面是否有警告或未解析交易
