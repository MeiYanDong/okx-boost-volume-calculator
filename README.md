# OKX Boost 交易量计算器

这份说明是给操作 Agent 用的。你的任务不是解释代码，也不是修改工具，而是帮助用户完成一次 OKX Boost 交易量计算，并把结果讲清楚。

## 你要完成什么

你要帮助用户回答一个问题：

用户的钱包在指定快照日期前后，BNB Chain 上的 OKX Boost 10 日平均交易量是多少？

工具会自动扫描 BNB Chain 上的 OKX DEX 聚合交易，并根据当前规则计算 Boost 交易量。

## 操作前先确认

开始前，先向用户确认这几项信息：

1. 钱包地址
2. 快照日期
3. 是否有某些代币存在活动额外加成

如果用户不知道额外加成，可以先不填。先跑一次结果，再根据页面识别出来的代币补充加成。

## 打开工具

让用户或你自己打开本地页面：

http://localhost:5173/

如果页面打不开，不要猜测计算结果。请告诉用户：工具服务可能还没有启动，需要先启动本地服务。

## 第一次计算怎么做

1. 在“钱包地址”里填入用户的钱包地址。
2. 在“快照日期”里选择用户关心的快照日期。
3. 点击“扫描 BNB OKX 聚合交易”。
4. 等待扫描完成。
5. 记录页面顶部的 Boost 交易量。

如果扫描过程中出现报错，先把报错原文告诉用户，不要自己编造原因。

## 快照日期怎么判断

快照日期不是“结束日期前一天”，而是被包含在统计窗口里的那一天。

如果选择 5 月 3 日，工具会站在 5 月 3 日这一天的角度，看包含 5 月 3 日在内的过去 10 个 UTC 日期。

如果选择今天，且今天还没结束，结果只是实时预估。你需要提醒用户：今天后面如果继续交易，结果还会变。

## 额外加成怎么处理

先跑一次扫描。扫描完成后，页面会出现“代币额外加成”区域。

如果某个代币有活动额外加成：

1. 找到这个代币。
2. 在它旁边填写加成百分比。
3. 例如额外加成是 50%，就填 50。
4. 填完后结果会自动更新。

不要因为填了某个代币加成，就只看这个代币的交易。额外加成只是倍率，不是筛选条件。所有被工具识别到的 BNB Chain OKX 聚合交易仍然会参与计算。

## 你需要向用户汇报什么

计算完成后，用简短的话告诉用户：

1. 当前 Boost 10 日平均交易量是多少
2. 10 日合计交易量是多少
3. 统计窗口是哪几天
4. 一共识别了多少笔交易
5. 是否用了代币额外加成
6. 是否有警告或未解析交易

如果用户是在判断是否达标，你还要明确告诉他：

- 当前结果是否已经高于目标线
- 如果没达标，还差多少
- 如果今天还没结束，要提醒这是实时预估，不是最终快照结果

## 重要边界

当前工具只统计 BNB Chain。

不要把下面这些交易算进去：

- Base 上的 OKX 交易
- Arbitrum 上的 OKX 交易
- Solana 上的 OKX 交易
- 其他链上的 OKX 钱包交易

如果用户说“OKX 钱包里明明有交易量，但工具没算到”，你要先让他提供交易 hash，然后判断这笔交易是不是 BNB Chain。

如果交易实际发生在 Base 或其他链，应该告诉用户：这不是当前工具漏算 BNB，而是当前版本暂未统计多链。

## 结果有小差异时怎么解释

如果工具结果和 OKX 页面差一点点，不要直接说谁错了。

你可以这样解释：

链上工具主要根据稳定币侧成交额估算 USD；OKX 页面可能使用自己的历史价格和内部口径，所以出现小数级差异是正常的。

如果差异很大，再继续排查：

1. 是否选错快照日期
2. 是否交易发生在其他链
3. 是否某些代币额外加成没填
4. 是否有交易解析失败的警告

## 什么情况下要停止并问用户

遇到下面情况，不要继续猜：

- 用户没有提供钱包地址
- 用户没有说明快照日期
- 页面报错但你看不懂原因
- 用户声称某笔交易被漏算，但没有提供交易 hash
- 用户要求统计 Base、Arbitrum、Solana 等当前版本不支持的链

这时应该向用户要具体信息，而不是继续给结论。

## 不要做什么

不要让普通用户填写 API Key。

不要让普通用户导入 CSV。

不要向用户解释内部实现。

不要把当前工具说成支持所有 OKX 钱包交易。

不要把实时预估说成最终结果。

不要忽略页面里的警告信息。
