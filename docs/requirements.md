# OKX Boost 钱包总览需求纲领

> 维护规则：本文是用户需求的唯一纲领。Agent 只能读取和引用本文；只有在用户明确要求“更新需求文档”时，Agent 才能修改本文。

## 目标用户

参与 OKX Boost 活动、需要让 Agent 帮自己计算多个钱包 Boost 交易量的非技术用户。

## 核心痛点

用户不是只想计算某一个钱包或某一个代币，而是想在首页直接看到所有钱包最近 10 天的 Boost 交易情况，并快速判断是否达标。

当前最需要解决的问题：

1. 多个钱包扫过后不能重复扫，除非用户强制要求。
2. 首页只展示最近 10 天的数据，包括今天。
3. 代币加成是二级信息，不能打断“查看所有钱包”的主流程。
4. 交易明细是二级信息，用户点击钱包后再看。
5. 技术配置项不能暴露给普通用户，例如 Ankr、Etherscan API Key、CSV。

## 产品形态

首页应是一个“钱包 Boost 运营工作台”，不是单钱包计算器。

首页第一屏必须包含：

1. 左侧导航。
2. 顶部应用栏。
3. 顶部工具条：快照时间、最近 10 天、钱包筛选、归档状态、强制重扫、刷新。
4. 中央主表：所有钱包的 Boost 总览。
5. 右侧总览：所有钱包的聚合状态和达标进度。

代币与交易明细只能作为二级信息出现。

## 视觉方向

布局复刻用户提供的后台工作台参考图。

主题色不使用 OKX 交易所风格；使用当前项目的 OKX Wallet 风格：

1. 黑灰主背景。
2. 白色高对比文字。
3. Boost 绿色作为核心强调色。
4. 低饱和边框和状态色。

## 数据规则

快照日期包含当天。

统计窗口为快照日期向前 10 个 UTC 日期，包括快照日期当天。例如快照日期是 2026-05-03，则窗口为 2026-04-24 UTC0 至 2026-05-04 UTC0 之前。

如果快照日期是今天且今天未结束，结果是实时预估。

## 归档规则

已经扫描过的钱包结果应保存在本地归档中。

普通同步优先读取归档，不重复扫描链上数据。

只有用户点击强制重扫时，才重新扫描链上数据。

## 当前边界

当前版本只统计 BNB Chain 的 OKX DEX Router 交易。

Base、Arbitrum、Solana 和其他链暂不纳入本版本。
