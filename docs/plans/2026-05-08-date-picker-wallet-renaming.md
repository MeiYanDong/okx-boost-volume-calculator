# S9 子 Plan：日期选择与钱包重命名

## 阶段目标

修复主工具栏日期不可直接选择的问题，并让用户能给钱包自定义名称，降低多钱包场景下只看短地址的识别成本。

## 范围

1. 日期控件从隐藏 input 改回可见、可点击、可输入的原生 date input。
2. 钱包列表支持「名称 地址」格式，旧的一行一个地址格式继续兼容。
3. 钱包表格增加「重命名」操作。
4. 钱包详情页增加「重命名」操作。
5. 扫描记录、统计报表、详情标题优先展示自定义钱包名称。

## 不做

1. 不改快照日期的 UTC 统计逻辑。
2. 不改交易扫描、归档、增量刷新或计算逻辑。
3. 不引入独立钱包管理数据库，名称仍保存在本机浏览器的钱包列表文本中。

## 验收标准

1. 主日期控件在浏览器中是可见且可操作的 date input。
2. 钱包列表填写 `主钱包 0x...` 后，表格展示 `主钱包`。
3. 表格行可点击「重命名」更新钱包名称。
4. 详情页可点击「重命名」更新钱包名称。
5. `npm run lint` 与 `npm run build` 通过。
