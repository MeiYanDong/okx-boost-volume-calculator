# S42 飞书每日快照日报

## 背景

用户反馈当天没有收到飞书推送。排查后确认飞书配置存在且启用，Cron 时间为北京时间 08:05 附近，问题在通知判断：旧逻辑只在未来快照有风险或刷新失败时发送，全部安全时不会推送。

## 实现

1. Cron API 的发送判断改为：只要工作区配置了飞书 Webhook，就发送每日快照日报。
2. 风险和失败保留为日报状态，不再作为是否发送的唯一条件。
3. Cron 生成的通知正文从“预警”改为“日报”，安全状态也输出。
4. 日报包含当前快照、自动刷新成功数、未来 3 天快照摘要、到期交易量和最大差额。

## 验收

- `npm run build` 通过。
- 本地 dry run 只指定用户 Supabase workspace，返回 `shouldNotify: true`。
- dry run 预览正文为“OKX Boost 自动快照日报”，状态为“当前无风险”，包含 2026-05-14 至 2026-05-17 北京时间快照摘要。
