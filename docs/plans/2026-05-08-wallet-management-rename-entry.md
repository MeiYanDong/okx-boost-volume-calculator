# S12 子 Plan：钱包管理页重命名入口

## 阶段目标

让钱包管理页本身承担完整的钱包配置能力，补齐显式「重命名」入口，避免用户只能在总览表格或详情页里发现重命名功能。

## 范围

1. 在钱包管理页「当前钱包」列表中为每个钱包增加 `重命名` 按钮。
2. 复用现有 `renameWallet` 逻辑，继续写回钱包源文本。
3. 按钮带图标、hover 反馈和移动端换行规则。
4. 不改变钱包文本格式、不改扫描逻辑。

## 验收标准

1. 钱包管理页的每个钱包行都能看到 `重命名`。
2. 点击后可以调用现有重命名流程。
3. 桌面端按钮不挤压状态标签。
4. 移动端按钮换行展示，不产生横向溢出。
5. `npm run lint` 与 `npm run build` 通过。
