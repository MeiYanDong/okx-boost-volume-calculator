# S25 子 Plan：取消全局飞书 Webhook

## 阶段目标

飞书 Webhook 属于用户个人通知通道，不作为项目全局资源。系统只允许登录用户使用自己保存的飞书机器人；未登录用户、旧数据空间和未配置个人 Webhook 的工作区都不发送飞书。

## 实现边界

1. `/api/feishu` 必须要求登录用户。
2. `/api/feishu` 不再接受私有访问码触发全局 Webhook。
3. Cron 只对 Supabase 工作区读取所属用户的个人飞书配置。
4. Cron 遇到未配置个人 Webhook 的工作区时跳过通知，不回退到全局环境变量。
5. `发送测试` 只验证连通性，不发送钱包地址、交易哈希或交易量。
6. 文档移除全局飞书 Webhook 配置方式。

## 验收标准

1. 未登录请求 `/api/feishu` 返回 401。
2. 登录但未配置个人 Webhook 时 `/api/feishu` 返回未启用提示。
3. 登录且配置个人 Webhook 时可发送测试消息。
4. 测试消息不包含真实钱包或交易数据。
5. Cron 未配置个人 Webhook 时不发送通知。
6. README 和部署文档不再引导配置全局飞书 Webhook。

## 当前状态

已完成。代码、文档和 Vercel 生产部署均已更新。线上 smoke test 覆盖未登录拒绝、私有访问码不兜底、临时登录用户未配置个人 Webhook 时拒绝发送，并已清理测试账号与邀请码。
