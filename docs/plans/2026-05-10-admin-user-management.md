# S23 子 Plan：管理员用户管理

## 阶段目标

把邀请制账号系统补齐为可运营的用户管理闭环。管理员不仅能创建邀请码，还能看到已注册用户，调整钱包额度，并在必要时禁用或重新启用账号。

## 实现边界

1. 后端 `/api/auth` 新增管理员动作：`list-users`、`update-user`。
2. 用户列表来自 `app_profiles`，附带工作区数量和已启用钱包数量。
3. 管理员可更新用户 `max_wallets`，范围为 1 到 500。
4. 管理员可把用户状态切换为 `active` 或 `disabled`。
5. 禁止禁用当前登录管理员账号。
6. 禁止禁用最后一个 active 管理员账号。
7. 本阶段不做角色升降级，不做删除用户，避免高风险误操作。

## 验收标准

1. 管理员登录态可以读取用户列表。
2. 管理员可调整普通用户钱包额度，保存后列表展示新额度。
3. 管理员禁用普通用户后，该用户不能再访问受保护扫描接口。
4. 管理员可重新启用普通用户。
5. 禁用当前管理员或最后一个 active 管理员会被服务端拒绝。
6. 普通用户不能调用用户管理接口。

## 当前状态

代码实现和基础类型检查已完成，等待真实 Supabase 权限链路测试、构建、提交、推送和生产部署。

## 后续建议

1. 增加用户级飞书通知配置。
2. 增加用户用量统计和每日刷新/重扫真实限制。
3. 增加管理员操作日志，记录谁改了谁的额度或状态。
