# S20 子 Plan：管理员邀请码管理页

## 阶段目标

把邀请码管理从隐藏 API 操作升级为页面内可完成的管理员工作流，降低私人部署后的日常运维成本。

## 实现边界

1. 后端 `/api/auth` 增加邀请码列表和撤销动作，仍然使用私有访问码或 `CRON_SECRET` 做管理员鉴权。
2. 服务端返回脱敏邀请码元数据，不返回 `code_hash`，邀请码原文只在创建成功后展示一次。
3. 前端偏好设置页增加管理员私有访问码、创建邀请码、刷新列表和撤销未使用邀请码。
4. 邀请码列表展示邮箱、创建时间、钱包额度、过期时间和使用状态。
5. 设置页保持响应式布局，中宽屏和窄屏不产生隐式列或挤压。

## 验收标准

1. 管理员可在页面生成邀请码。
2. 新邀请码原文只在创建后展示一次。
3. 管理员可读取最近邀请码列表。
4. 未使用的邀请码可撤销，已使用或过期的邀请码不能撤销。
5. 前端类型检查、服务端语法检查和生产构建通过。
6. 本地真实 Supabase 管理 API 测试通过。

## 当前状态

已完成后端接口、前端设置页入口、响应式样式、文档更新和构建验证。

## 后续建议

1. 将飞书 Webhook 改为用户级设置，避免多用户共用全局通知。
2. 增加管理员用户角色判断，逐步减少对全局私有访问码的依赖。
3. 记录邀请码管理操作日志，方便排查邀请来源。
