# S33 扫描结果立即云端保存

## 目标

扫描或增量刷新成功后，结果必须尽快写入当前账号的 Supabase 云端归档；未登录旧路径则写入服务端数据空间。不能只依赖后台延迟同步，避免用户扫描完成后立刻刷新导致结果看似丢失。

## 实现

1. 单个钱包扫描成功后构建当前钱包结果快照。
2. 立即调用归档保存接口，登录态走 Supabase，私有访问码走服务端归档。
3. 保留原有后台同步作为兜底。
4. 保存状态文案区分 Supabase 云端和服务端数据空间。

## 验收

1. 扫描成功后无需等待下一轮后台 effect，即可发起归档写入。
2. 已登录用户保存到 Supabase。
3. 未登录但有私有访问码的旧路径仍保存到服务端归档。
