# S26 子 Plan：飞书真实数据测试发送

## 阶段目标

把“发送测试”从纯机器人连通性检查升级为真实生产链路演练。测试时读取当前登录用户的 Supabase 云端归档，执行一次每日自动刷新逻辑，生成真实预测摘要，并发送到该用户自己的飞书机器人。

## 实现边界

1. `/api/feishu` 支持 `mode: "real-data-test"`。
2. 真实数据测试必须要求登录用户和个人飞书 Webhook。
3. 测试读取当前用户默认工作区归档，不读取全局数据空间。
4. 测试复用每日自动刷新逻辑，包括 RPC 扫描、加成规则和未来快照预测。
5. 测试消息必须标记为测试，并说明不会写入归档。
6. `api/feishu.mjs` 在 Vercel 上使用 300 秒函数时长，避免多钱包演练被默认超时截断。
7. 当 `BSC_RPC_URL` 为空但已配置 Ankr Multichain 时，自动推导 Ankr BSC 标准 RPC，避免回退到历史区块裁剪的公共 RPC。
8. Ankr URL 统一规范化，只取链名后的 token，忽略尾部误粘贴的 `/n` 等残留路径段。

## 验收标准

1. 未登录用户不能触发真实数据测试。
2. 未配置个人 Webhook 的登录用户不能触发真实数据测试。
3. 已配置个人 Webhook 的用户点击测试时，服务端读取真实归档并发送预测摘要。
4. 测试不会保存 `updatedArchive`，不改变归档数据。
5. README 和设置页文案明确“真实数据测试”的含义。

## 当前状态

已完成。代码、文档、本地构建验证、真实归档演练、Vercel 生产部署和线上 smoke test 均已完成。真实归档演练覆盖 6 个钱包，修复 Ankr URL 规范化后结果为 6 成功、0 失败；线上验证覆盖未登录拒绝和登录但未配置个人 Webhook 时拒绝发送。
