# S11 子 Plan：钱包管理页布局重构

## 阶段目标

使用前端设计 Skill 重构「钱包管理」页面，让它成为独立的钱包配置工作台，而不是普通表单页。

## 范围

1. 钱包管理页不再展示全局扫描工具条。
2. 页面顶部增加钱包源状态条，展示有效地址、归档、待扫描、失败、访问码状态。
3. 主区域左侧改为大编辑器，专门维护「名称 地址」源数据。
4. 主区域右侧展示归档进度和当前钱包预览。
5. 刷新新增交易和强制重扫保留在钱包管理页内部。
6. 移动端布局改为纵向堆叠，避免横向溢出。
7. 钱包管理页不再额外重复页面标题，顶部标题之后直接进入工作区。

## 不做

1. 不改变钱包数据存储方式。
2. 不改变扫描、归档、增量刷新、强制重扫和计算逻辑。
3. 不恢复任何其他页面的钱包编辑入口。

## 验收标准

1. 钱包管理页可以独立完成地址维护、访问码填写和刷新动作。
2. 钱包管理页不显示日期筛选、钱包筛选等全局扫描工具条。
3. 桌面端主编辑区和右侧状态区层级清晰。
4. 移动端没有横向溢出。
5. 页面内没有重复的钱包管理标题卡。
6. `npm run lint` 与 `npm run build` 通过。
