# S6 子 Plan：行动驱动交互改良

## 阶段目标

把首页从“很多按钮同时出现”的数据看板，改成“用户一眼知道下一步点哪里”的行动工作台。

## 问题判断

原交互的问题不是单个按钮样式，而是动作层级混乱：

1. 全局刷新、实时更新、强制重扫、钱包列表、导入钱包同时出现在首屏。
2. 危险操作 `强制重新扫描` 暴露在顶部主路径。
3. 同一个入口有多个名字，例如 `钱包列表` 和 `导入钱包`。
4. 钱包行没有按状态给出唯一下一步。
5. hover、active、focus 状态不足，鼠标悬浮和点击反馈生硬。

## 设计决策

1. 顶部工具条只保留一个动态主动作。
2. 主动作按状态变化：添加钱包、扫描待处理钱包、重试失败钱包、刷新新增交易、扫描中。
3. 强制重扫全部收进 `更多` 菜单，并加二次确认。
4. 钱包管理统一命名为 `钱包列表`。
5. 钱包行按状态展示动作：未扫描为扫描，失败为重试，已归档为详情和刷新新增。
6. 已归档钱包整行可点击打开详情。
7. 所有主要按钮、导航、行、输入框补齐 hover、active、focus-visible 反馈。
8. 钱包列表面板不再使用内部关闭叉号，由触发它的 `钱包列表` 按钮再次点击收起。

## 当前状态

已完成。首页主路径已经收敛为一个动态主按钮；危险操作被隔离；钱包行有状态化动作；hover 和点击反馈已覆盖导航、工具条、表格行、行内按钮、输入框和更多菜单。钱包列表面板已改为按钮 toggle，不再要求用户把鼠标移到面板内部的关闭按钮。

## 验收标准

1. 顶部不再展示无文字刷新图标。
2. 顶部不再直接展示 `强制重新扫描`。
3. 首屏只有一个全局主动作按钮。
4. `强制重扫全部` 在更多菜单里，并需要二次确认。
5. `导入钱包` 统一改为 `钱包列表`。
6. 钱包失败时展示 `重试`。
7. 已归档钱包展示 `详情` 和 `刷新新增`。
8. 已归档钱包整行 hover 有反馈，点击可打开详情。
9. 主要按钮 hover / active / focus-visible 有明确反馈。
10. 钱包列表面板无内部关闭叉号。
11. 点击 `钱包列表` 可展开，再次点击可收起。
12. `npm run lint` 通过。
13. `npm run build` 通过。
14. 浏览器验证首屏按钮数量和交互状态符合预期。
