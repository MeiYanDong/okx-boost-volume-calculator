# S19 子 Plan：邀请制登录与 Supabase 云端归档

## 阶段目标

把 S18 的 Supabase 数据层接入真实应用，让不同用户用账号隔离数据，而不是依赖手填数据空间码。

## 实现边界

1. 新增 `/api/auth`，支持登录、刷新 session、邀请码注册和管理员创建邀请码。
2. 新增服务端 Supabase 封装，前端不直接连接 Supabase，不暴露 secret key。
3. `/api/archive` 支持双路径：登录用户走 Supabase，未登录用户继续走旧 Upstash 数据空间。
4. 前端顶部增加账号入口；钱包管理页在登录状态下显示云端账号，不再要求数据空间码。
5. Cron 同时兼容 Supabase 工作区和旧 Upstash 工作区，避免迁移期丢自动刷新。

## 验收标准

1. 未登录且未填写私有访问码时，不自动打 `/api/archive` 产生 401 噪声。
2. 管理员可用私有访问码创建邀请码。
3. 用户可用邀请码注册并自动登录。
4. 登录用户保存钱包归档后，刷新页面可以从 Supabase 恢复。
5. 旧 Upstash 私有访问码路径仍可使用。
6. 定时刷新接口能识别 Supabase 工作区。

## 当前状态

已完成代码接入、真实 Supabase 认证/归档 API 测试、浏览器 UI 验证和文档更新。

## 后续建议

1. 做管理员邀请码管理页，避免只靠 API 创建邀请码。
2. 将当前 default Upstash 归档迁移到管理员 Supabase 账号。
3. 按用户配置独立飞书 webhook，而不是全部使用全局 webhook。
