# S14 子 Plan：清理归档入口移除与飞书通知

## 阶段目标

移除用户不需要的“清理归档”入口，避免误删本地归档；同时把快照预测风险接入飞书群机器人提醒。

## 实现边界

1. 不做导入导出。Vercel 稳定生产链接不会因为更新代码而变化，数据连续性优先依赖同一浏览器本地归档。
2. 不在前端展示 Ankr、Chainstack、Etherscan 或飞书密钥。
3. 飞书通知只作为快照预警卡片里的动作，不新增单独提醒设置页面。
4. 飞书 Webhook 只从服务端环境变量读取。

## 当前状态

已完成。侧边栏和偏好设置不再展示清理归档入口；前端在未来快照预测出现低于目标的钱包时展示“飞书提醒”按钮；后端新增 `/api/feishu`，由 `ACCESS_PASSWORD` 保护，并限制 Webhook 只能指向飞书官方机器人地址。

## 验收标准

1. 页面没有“清理归档”或“清除归档”入口。
2. 快照预测无风险时不展示飞书提醒按钮。
3. 快照预测有风险时可点击飞书提醒按钮。
4. `/api/feishu` 未配置 Webhook 时返回明确错误，不影响主流程。
5. `/api/feishu` 支持飞书机器人签名密钥。
6. `.env.example` 和 Vercel 部署文档说明飞书环境变量。
7. `npm run build` 通过。
