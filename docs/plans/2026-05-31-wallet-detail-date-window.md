# S47 钱包详情每日数据窗口口径修复

## 目标

钱包详情页的「每日数据」必须和顶部当前快照日保持一致：快照日及前 9 个 UTC 日期，共 10 天。不能因为 Supabase 返回旧归档或结果对象自带旧窗口，就在详情页展示错误日期范围。

## 实现

1. 页面初始化和 Supabase 归档恢复继续使用当前 UTC 快照日，不用归档 `endDate` 覆盖页面快照日。
2. 钱包详情抽屉接收当前 `endDate`。
3. 详情展示前按当前 `endDate` 重新构造 10 日窗口。
4. 每日数据补齐窗口内无交易日期，并只展示窗口内日期。
5. 代币加成项和交易明细同步过滤到当前窗口，避免二级信息与每日数据日期范围不一致。

## 验收

1. 刷新网页后，顶部快照日仍为当前 UTC 日。
2. 打开任意钱包详情，「每日数据」范围等于顶部快照日及前 9 天。
3. 切换顶部快照日后，详情页每日数据、代币加成和交易明细同步切换窗口。
4. `npm run lint` 通过。
