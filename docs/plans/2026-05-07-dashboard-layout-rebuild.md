# S1 子 Plan：Dashboard Layout Rebuild

## 阶段目标

把当前页面从“输入表单 + 结果卡片”的计算器形态，重构为参考图所示的钱包 Boost 工作台。

## 当前状态

已完成。代码布局已重构，lint、build 和浏览器验收已通过。

必须复刻参考图的信息布局：

1. 左侧固定导航栏。
2. 顶部应用栏。
3. 顶部工具条。
4. 中央所有钱包表格。
5. 右侧总体概览栏。
6. 代币与交易明细作为二级入口。

主题色继续沿用当前项目的 OKX Wallet 黑灰 + Boost 绿色风格。

## 当前问题

当前实现已经有多钱包归档、强制重扫、详情区和 OKX Wallet 深色主题，但布局仍然是计算器页面：

1. 输入区占据首屏核心位置。
2. 指标卡在表格之前，削弱了“所有钱包总览”的主体地位。
3. 没有左侧导航。
4. 没有右侧固定总览栏。
5. 顶部工具条不接近参考图。

因此本阶段不是小修 CSS，而是重构页面骨架。

## 信息架构

页面结构调整为：

```text
AppFrame
  Sidebar
  MainWorkspace
    Topbar
    Toolbar
    ContentGrid
      WalletTablePanel
      RightOverviewRail
    WalletDetailDrawer / WalletDetailPanel
```

## 功能边界

本阶段要做：

1. 复刻参考图布局。
2. 保留现有多钱包归档逻辑。
3. 保留普通同步读取归档、强制重扫重新扫描的行为。
4. 保留钱包详情里的每日数据、代币加成、交易明细。
5. 让首页第一屏重点展示所有钱包表格。

本阶段不做：

1. 不新增多链支持。
2. 不引入后端数据库。
3. 不改 OKX Boost 核心计算公式。
4. 不公开展示 Ankr、Etherscan API Key、CSV。
5. 不把左侧导航做成完整多页面应用。

## 布局要求

### 左侧导航

固定宽度，参考图结构：

1. 总览
2. 钱包管理
   - 所有钱包
   - 分组管理
   - 导入钱包
3. 数据与扫描
   - 扫描记录
   - 扫描任务
4. 分析
   - 统计报表
5. 设置
   - 提醒设置
   - 偏好设置

当前阶段可以只让“总览”高亮，其余作为占位入口。

### 顶部应用栏

包含：

1. 产品标识：OKX Boost 钱包总览。
2. 右侧图标：帮助、通知、用户。

不需要真实账号系统。

### 顶部工具条

包含：

1. 快照时间。
2. 最近 10 天（含今天）的窗口选择。
3. 钱包筛选。
4. 归档状态提示。
5. 强制重新扫描按钮。
6. 刷新按钮。

当前已有的钱包地址输入可以降级到导入/管理区域，不再占据首屏主视觉。

### 中央钱包表

表格字段：

1. 钱包地址 / 备注名。
2. 状态。
3. 10 日合计 Boost。
4. 10 日平均 Boost。
5. 今日 Boost。
6. 目标差额。
7. 最后扫描。
8. 操作。

钱包表是首页主体。

### 右侧总体概览

包含：

1. 钱包总数。
2. 已归档钱包。
3. 扫描中钱包。
4. 等待扫描钱包。
5. 10 日合计 Boost。
6. 10 日平均 Boost。
7. 今日 Boost。
8. 目标达成率。

右侧下方保留“代币与交易明细”入口。

## 交互要求

1. 点击同步钱包归档：普通同步，优先使用本地归档。
2. 点击强制重新扫描：重新扫描链上数据。
3. 点击查看详情：打开二级详情，不影响首页总览。
4. 修改代币加成：只做本地试算，不重新扫链。
5. 刷新页面后：已归档数据自动恢复。

## 验收标准

1. 首页第一屏能直接看到所有钱包表格。
2. 左侧导航、顶部栏、工具条、中间表格、右侧概览五块区域清晰存在。
3. 布局接近用户参考图，不再像计算器表单页。
4. 颜色使用 OKX Wallet 黑灰 + Boost 绿色。
5. `npm run lint` 通过。
6. `npm run build` 通过。
7. 浏览器打开 `http://127.0.0.1:5173/` 后无明显重叠、溢出或空白主区域。
