# S7 子 Plan：网页端布局清理

## 阶段目标

修复网页端首屏暴露出的明显设计问题，让首页回到「直接查看所有钱包 Boost 状态」的主任务，不被钱包编辑面板、日期控件和重复入口干扰。

## 范围

1. 优化顶部日期控件，避免「最近 10 天」和日期输入挤在一起。
2. 钱包列表在桌面端改为浮层展示，不再把总体概览和钱包表格整体向下挤。
3. 钱包列表继续使用同一个入口开关，移除额外关闭按钮。
4. 移除表格标题里的重复钱包列表入口。
5. 拆分总体进度的大数字，避免当前值和目标值组合后换行堆叠。

## 不做

1. 不改扫描、归档、增量刷新、加成计算逻辑。
2. 不重做整体视觉主题。
3. 不新增二级页面。

## 设计决策

1. 日期控件保留原生 date input 能力，但前台展示为紧凑的文字日期。
2. 钱包列表桌面端使用固定浮层，手机端保持正常文档流，避免小屏被遮挡。
3. 首页主表格只负责展示钱包结果，不再承担钱包配置入口。
4. 总体进度优先展示当前累计值，目标值降级为辅助说明。

## 验收标准

1. 桌面端日期控件宽度稳定，不再撑满顶部工具栏。
2. 打开钱包列表后，总体概览位置不发生下移。
3. 钱包列表中不再出现独立的叉号关闭按钮。
4. 钱包表格标题区域不再出现第二个「钱包列表」按钮。
5. 总体进度金额不再拆成多行大字。
6. `npm run lint` 与 `npm run build` 通过。
7. 浏览器验证桌面和移动端没有横向溢出。
