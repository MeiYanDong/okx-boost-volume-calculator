# S29 会话与快照窗口审查

## 目标

继续从多用户隔离、服务端访问权限和快照统计口径审查明显错误，避免登录用户绕过账号状态校验，也避免切换快照日期后继续展示旧归档窗口的统计结果。

## 实现边界

1. 服务 API 请求优先使用当前有效登录态；只有浏览器没有登录 session 时才回落到私有访问码。
2. 缺失账号资料或被禁用账号不能继续被视为有效登录。
3. 首页总览、钱包列表和统计报表统一按当前快照日倒推 10 天，从每日明细动态聚合。
4. 钱包详情仍展示该归档自身的扫描窗口，用于排查原始结果。

## 验收

- `npm run lint`
- `npm run build`
- 生产部署后验证首页、未授权 API、Cron 鉴权和 auth me 接口。
