# S41 交易明细完整性加固

## 背景

全量对照发现，部分钱包已经通过 Ankr 找到完整候选交易 hash，但逐笔解析阶段遇到 RPC 限流后只写入 warning，最终仍保存了缺少交易明细的归档。

## 实现

1. RPC 请求增加临时错误重试，覆盖 429、5xx、timeout、rate limit、fetch failed 等场景。
2. 单链扫描解析候选 hash 时收集解析错误；只要存在未解析完成的 hash，就抛错并阻止保存半成品。
3. 增量刷新只携带当前 10 日窗口内已解析交易的 hash，避免旧窗口 hash 污染完整性判断。
4. Supabase 写入前检查 `chainScans.txHashes` 与 `swaps` 的覆盖关系，拒绝交易明细不完整的新结果。
5. 历史归档本身不完整时，允许完整的新扫描结果覆盖旧残缺数据。
