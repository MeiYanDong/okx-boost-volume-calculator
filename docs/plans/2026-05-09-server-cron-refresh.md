# S16 子 Plan：服务端归档与每日自动刷新

## 阶段目标

把“浏览器本地归档”升级为“服务端归档 + 浏览器自动同步”。Vercel 生产环境每天北京时间 08:05 自动增量刷新钱包，刷新后预测当前快照和未来 3 次快照；如果任一钱包低于单钱包 10 日累计目标，自动发送飞书提醒。

## 实现边界

1. 不要求用户手动导入导出数据。
2. 钱包列表、扫描结果、加成规则、目标线保存到 Vercel 绑定的 Upstash Redis。
3. 前端仍保留浏览器本地缓存，服务端归档是跨部署和定时任务的数据源。
4. 定时任务只做增量刷新；强制重扫仍由用户在页面手动触发。
5. 飞书通知只在风险或自动刷新失败时发送，不发送“全安全”噪音。
6. Cron 快照口径为：北京时间 08:05 运行，确认上一 UTC 日的快照。

## 实现方案

1. 新增 Upstash Redis 归档层，保存钱包列表、结果、加成规则和扫描历史。
2. 新增 `/api/archive`，由 `ACCESS_PASSWORD` 保护，前端启动时恢复服务端归档，状态变化后自动同步。
3. 新增 `/api/cron/daily-refresh`，由 `CRON_SECRET` 或 `ACCESS_PASSWORD` 保护。
4. 新增 Vercel Cron：`5 0 * * *`，即 UTC 00:05 / 北京时间 08:05。
5. 服务端复用现有计算器，按 `scannedToBlock` 增量刷新新区块，并重新汇总滚动 10 日窗口。
6. 自动刷新完成后生成当前快照和未来 3 次快照预测，并在风险存在时调用飞书机器人。

## 当前状态

已完成本地实现，待完成生产部署与线上 smoke test。

## 验收标准

1. `npm run lint` 通过。
2. `npm run build` 通过并生成 `.server/cronJob.mjs`。
3. `/api/archive` 支持读取和写入服务端归档。
4. `/api/cron/daily-refresh?dryRun=1` 支持 dry-run，不真实发送飞书。
5. Vercel `vercel.json` 含每日 Cron 配置。
6. 生产环境已配置 Upstash Redis 和 `CRON_SECRET`。
7. 生产部署完成后，稳定链接不变。
