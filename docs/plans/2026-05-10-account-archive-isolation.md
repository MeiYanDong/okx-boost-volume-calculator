# S27 子 Plan：账号归档隔离加固

## 阶段目标

修复同一浏览器里注册或登录新账号后，新账号继承旧账号/本机钱包列表的问题。Supabase 登录用户的数据必须以云端归档为准；浏览器本地缓存只能作为未登录或旧数据空间模式的便利缓存。

## 根因

新账号的 Supabase 云端归档为空时，前端仍保留同一浏览器里的本地 `walletsText`。随后自动同步逻辑把这份本地钱包列表保存到了新账号默认工作区，导致新用户看到初始化钱包配置。

## 实现边界

1. Supabase 登录用户读取云端归档时不再与浏览器本地钱包列表合并。
2. 登录账号的云端归档为空时，前端重置为空工作区，而不是保存当前本地数据。
3. 自动同步必须确认当前账号归档加载完成，不能用上一个账号的 ready 状态触发保存。
4. 登录账号扫描时不读写全局本地 result cache，避免同浏览器账号之间复用链上结果。
5. 登录账号的扫描历史不写入全局本地 scan history。
6. 未登录和旧数据空间模式继续保留本地缓存便利。

## 验收标准

1. 同一浏览器里注册新账号，云端无钱包时页面显示空钱包列表。
2. 新账号空归档不会自动保存旧钱包列表。
3. 登录已有账号时，以 Supabase 云端钱包列表为准，不合并本地钱包。
4. 账号切换过程中，自动同步不会在当前账号归档加载完成前执行。
5. 未登录模式仍可继续使用本地缓存。

## 当前状态

已完成。代码修复、文档更新、本地 lint/build、Vercel 生产部署和线上 smoke test 均已完成。线上临时新账号验证结果为 `walletsTextLength=0`、`recordsLength=0`，未泄露初始化钱包。
