# S34 Supabase 归档读取瘦身

## 目标

刷新页面恢复云端归档时，不再一次性读取工作区全部历史扫描结果的大 JSON。首页只需要每个钱包当前快照日或最近一条可用结果。

## 实现

1. 先读取 `wallet_scan_results` 的轻量元数据，不包含 `result`。
2. 按钱包选择当前 UTC 快照日结果；如果没有，则选择最近结果。
3. 第二次查询只读取被选中的结果 JSON。
4. 保持历史结果仍留在 Supabase，不删除数据。

## 验收

1. `myandong1@gmail.com` 工作区仍能恢复 8 个钱包。
2. 读取完整 JSON 的数量从全部历史结果收敛到每个钱包最多 1 条。
3. 构建验证通过。
