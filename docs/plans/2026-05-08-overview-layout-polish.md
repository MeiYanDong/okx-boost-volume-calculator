# S5 子 Plan：总体概览布局抛光

## 阶段目标

修复首页总体概览数值拥挤的问题，让用户第一眼先判断所有钱包是否接近 10 日累计目标，再进入钱包表格处理具体地址。

## 边界

本阶段只调整首页总体概览的信息结构和响应式布局，不改扫描逻辑、归档逻辑、加成规则和详情抽屉。

## 设计决策

1. 总体概览不再使用“卡片里再塞多张卡片”的结构。
2. 横向分成三个功能区：主结果、目标进度、状态摘要。
3. 主结果只放 `10 日合计 Boost` 和目标差额。
4. 目标进度只放单钱包目标输入、等效日均和达成率进度条。
5. 状态摘要只放四个只读指标，作为辅助扫描信息。
6. 代币与交易明细仍然留在单钱包详情里，不回到首页右侧。

## 当前状态

已完成。总体概览已改为一个横向工作台分区，宽屏下三栏展示，窄屏下按主结果、目标进度、状态摘要纵向折叠。数值不再挤在右侧窄栏，也不再以多张小卡片堆叠展示。

## 验收标准

1. 总体概览位于钱包表格上方。
2. 总体概览宽屏下展示主结果、目标进度、状态摘要三个区域。
3. `10 日合计 Boost` 是最醒目的数值。
4. 单钱包 10 日累计目标仍可直接编辑。
5. 达成率进度条仍可见。
6. 状态摘要不再使用四张挤压的小卡片。
7. 代币与交易明细不出现在首页右侧。
8. `npm run lint` 通过。
9. `npm run build` 通过。
10. 浏览器验证桌面和窄屏布局没有数值重叠。
